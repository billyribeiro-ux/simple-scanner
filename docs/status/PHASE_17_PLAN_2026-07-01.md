## Scope

Phase 17 is a live operator verification phase, not a new feature-expansion phase. Code changes are allowed only if live execution exposes a real parsing, redaction, entitlement-classification, timestamp, idempotency, freshness-gate, export, scheduler, or UI workflow defect.

The platform remains a scanner, research, validation, backtest, signal, and export system only. This phase will not add broker execution, order routing, automatic model activation, production WebSocket ingestion, options analytics, market internals, self-learning claims, or profitability claims.

## Runtime Secret Gate

`FMP_API_KEY` must be loaded only from the runtime shell, ignored local env files, or an existing OS secret mechanism. The key must not be printed, logged, committed, written to docs, stored in database rows, included in exports, exposed to frontend code, embedded in URLs, or copied into scheduler payloads/events.

Before live checks:

- confirm `.env` and `.env.local` are ignored;
- confirm git status does not include env files;
- confirm `make doctor` reports only key present or missing;
- run secret scans after any provider, export, scheduler, frontend, or docs work.
