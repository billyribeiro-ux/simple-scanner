# Phase 21R Plan - 2026-07-04

`PHASE_21R_STATUS = PLANNED_REPAIR_ONLY`

Phase 21R is an evidence-store restoration and regression-isolation hardening phase. It is not a model-improvement phase, not a feature-expansion phase, not a governance-gate loosening phase, and not a model-activation phase.

## Guardrails

- Do not activate, approve, or deploy the rejected challenger.
- Do not use contaminated DB rows as approval evidence.
- Do not add broker execution, order routing, options, market internals, Level 2, dark-pool data, production WebSocket ingestion, black-box ML, or self-learning behavior.
- Do not claim profitability.
- Do not expose `FMP_API_KEY` or put API keys in query strings.
- Do not silently delete or reset evidence data.

## Repair Plan

1. Define a database-role contract:
   - `DATABASE_URL` plus `AMD_DB_ROLE=evidence` for live evidence.
   - `TEST_DATABASE_URL` plus `AMD_DB_ROLE=test` for mutating regression tests.
2. Update Postgres API smoke and repository parity tests to use `TEST_DATABASE_URL` for fixture writes.
3. Add evidence-mode fixture guardrails so known fixture IDs such as `parity-*`, `test-*`, `smoke-*`, `fixture-*`, and `parity-model-accepted` cannot be inserted into evidence mode unless `AMD_ALLOW_TEST_FIXTURES_IN_EVIDENCE=true`.
4. Add tests proving fixture writes fail in evidence mode and pass in test mode.
5. Add reproducible DB audit tooling and reports for fixture/live row counts.
6. Preserve the contaminated database as an audited artifact unless a clean backup or regeneration path is explicitly documented.
7. Restore or regenerate clean evidence only if a safe source exists; otherwise report the exact blocker.
8. Re-run backend, frontend, database, export, and secret-scan checks without contaminating evidence.

## Expected Outcome

The best acceptable outcome is `PHASE_21R_STATUS = ACCEPTED` if test isolation and guardrails are fixed, the evidence store is clean/restored/regenerated, and tests pass without contamination. If test isolation is fixed but evidence cannot be restored because no clean backup/runtime data source is available, the correct status is `PHASE_21R_STATUS = PARTIAL_BLOCKED`.
