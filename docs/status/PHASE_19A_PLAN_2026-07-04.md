# Phase 19A Plan - Execution Evidence Reconciliation

Date: 2026-07-04

Phase 19A is an audit and execution-evidence reconciliation phase. It is not a new build phase and does not add features, trading behavior, broker execution, order routing, automatic model activation, WebSocket production ingestion, black-box ML, or profitability claims.

## Objective

Determine whether Phase 19 can be certified from real repository/runtime evidence, not from a report-pack artifact alone.

Phase 19A must reconcile:

- git branch, HEAD, commit history, and working tree state;
- runtime/database state for SQLite and Postgres where available;
- dirty-window before/after evidence;
- feature, candidate, label, and replay rebuild evidence;
- freshness before/after evidence;
- strict research-cycle dry-run evidence;
- export metadata, file hashes, source IDs, and artifact versions;
- scheduler rebuild job behavior;
- test and secret-scan results.

## Evidence Rules

- Do not invent missing runtime values.
- Prefer committed Phase 19 docs only as documentary evidence, not as a substitute for missing runtime rows or ignored export files.
- Do not reset or destroy local database volumes during evidence collection.
- Do not call FMP unless a bounded refresh is explicitly required and `FMP_API_KEY` is supplied only through runtime environment.
- Do not run `allow_stale=true` except as a labeled diagnostic if strict research dry-run blocks.
- If required runtime evidence is missing, mark `PHASE_19_STATUS = EVIDENCE_PENDING`.
- If strict research-cycle still blocks with complete rebuild evidence, mark `PHASE_19_STATUS = PARTIAL_BLOCKED`.
- If rebuild evidence is complete and strict dry-run passes, mark `PHASE_19_STATUS = ACCEPTED`.

## Planned Checks

1. Read Phase 18 and Phase 19 docs plus the artifact-readiness, scheduler, research, replay, API, repository, feature, candidate, label, and schema code paths.
2. Collect git status, branch, HEAD, recent log, diff stat, and `git diff --check`.
3. Inspect local runtime artifacts under ignored `data/`, `exports/`, and `model_artifacts/`.
4. Run `make doctor`, Docker compose config/up/ps, migration, DB inspection, and query diagnostics where practical.
5. Run backend, frontend, scheduler, export, FMP mocked, replay, Phase 19, compile, and e2e gates where practical.
6. Run targeted secret scans over source, docs, generated frontend output, exports, data, and model artifacts.
7. Create `PHASE_19A_COMPLETION_2026-07-04.md` with the final certification status and exact blockers.
