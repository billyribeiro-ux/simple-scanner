# Phase 19C Plan - Alembic Repair, Runtime Alignment, and Real-Data Regeneration

Date: 2026-07-04

Status: IN_PROGRESS

Phase 19C is a runtime repair and evidence regeneration pass. It does not add trading, order routing, options workflows, production WebSockets, model activation, self-learning, profitability claims, or any bypass around stale-data gates.

## Mission

1. Preserve current evidence before repair.
2. Repair the Alembic chain so a clean local Postgres database can migrate from base to head.
3. Align the backend runtime target to Python 3.14.6 when that interpreter is available.
4. Route Redis away from occupied host port 6379 with a configurable compose host port.
5. Restore Postgres gates: migration, schema inspection, query diagnostics, API smoke, and repository parity.
6. Check for real market data rows and recover or regenerate only from approved FMP sources when `FMP_API_KEY` is present.
7. Rebuild features, candidates, labels, replay, freshness reports, and strict research dry-run artifacts only from runtime rows.
8. Certify Phase 19 only from persisted rows and generated exports, with blockers recorded when data or runtime prerequisites are absent.

## Boundaries

- No fabricated bars, quotes, labels, model evidence, replay rows, freshness rows, or exports.
- No secret exposure in logs or reports.
- No destructive database reset unless evidence preflight proves no recoverable runtime evidence or the operator explicitly confirms.
- No stale-data gate bypass.
- No activation of any model, proposal, or execution path.

## Acceptance States

- `ACCEPTED`: Postgres migration is repaired, Python 3.14.6 is aligned, real bars exist or are recovered, Phase 19 artifacts are rebuilt, freshness and strict dry-run outputs are recorded, activation remains disabled, and tests/scans pass.
- `PARTIAL_BLOCKED`: Runtime repair and artifact rebuilds complete, but strict `allow_stale=false` research dry-run blocks on freshness or data quality.
- `BLOCKED_NO_DATA`: Runtime repair completes, but no real bars are present and no approved key/source is available for bounded regeneration.
- `BLOCKED_INFRA`: Python, Postgres/Alembic, Redis, or required runtime repair cannot be completed.
