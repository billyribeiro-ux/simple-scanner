## Planned Scheduler Hardening

The Phase 13 scheduler is non-autonomous and safe, but `run_job` and `run_pending` execute inside the caller path. Phase 14 will add a bounded local worker path if the current implementation remains request-bound:

- Add nullable lease/audit columns to `scheduler_jobs` through Alembic revision `0010_phase14_scheduler_worker`.
- Keep queued-job execution operator-triggered through `make scheduler-worker-once`; no daemon, cron, infinite loop, background autostart, or self-scheduling.
- Add repository methods for lease acquisition, heartbeat, release, and stale lease recovery.
- Add scheduler service methods for one-shot worker execution and stale recovery.
- Record events such as `JOB_LEASED`, `JOB_HEARTBEAT`, `JOB_RELEASED`, and `JOB_STALE_RECOVERED`.
- Preserve existing API behavior for direct single-job and run-pending operations.
- Ensure scheduler paths never approve proposals, activate models, place trades, route orders, run live FMP production ingestion, or expose secrets.
