# Phase 11 Plan

Status date: 2026-07-01

Phase 11 adds a controlled daily research-review cycle and challenger proposal lifecycle. It must stay diagnostic and governance-focused: no broker execution, no order routing, no silent retraining, no silent activation, no self-learning language, and no profitability claims.

## Implementation Sequence

1. Add persistence first:
   - create Alembic revision `0008_phase11_research_governance`;
   - add `research_cycles`, `research_cycle_artifacts`, `champion_challenger_comparisons`, `model_proposals`, and `model_decision_ledger`;
   - mirror the same tables in SQLite bootstrap;
   - update expected table/revision/schema inspection constants.

2. Add repository APIs:
   - save/get/list/update research cycles;
   - persist cycle artifacts;
   - save/list champion-vs-challenger comparisons;
   - create/update/list model proposals;
   - append/query decision ledger events;
   - expose the repositories through `RepositoryRegistry`.

3. Add governance services:
   - `ResearchCycleService` to create, dry-run, run, list artifacts, and export cycles;
   - `ModelProposalService` to create proposal records, approve, reject, and explicitly activate approved proposals;
   - `ChampionChallengerComparisonService` to compare active champion and challenger evidence without mutating active model state;
   - `ResearchStatusService` to summarize latest cycles, pending proposals, drift, review status, stale windows, and data quality.

4. Keep orchestration bounded and reproducible:
   - default `allow_stale=false`;
   - default `refresh_data=false`;
   - block `refresh_data=true` when `FMP_API_KEY` is absent;
   - persist config hash, input fingerprint, git commit, database revision, backend, warnings, artifact IDs, and no secrets;
   - enforce `max_window_count`;
   - keep dry-run diagnostic and non-mutating except for optional dry-run cycle records.

5. Add API routes:
   - `POST/GET /research/cycles`;
   - `GET/POST /research/cycles/{research_cycle_id}`;
   - `POST /research/cycles/{research_cycle_id}/run`;
   - `POST /research/cycles/{research_cycle_id}/dry-run`;
   - `GET /research/cycles/{research_cycle_id}/artifacts`;
   - `POST /research/cycles/{research_cycle_id}/export`;
   - proposal list/get/approve/reject/activate routes;
   - `GET /research/decision-ledger`;
   - `GET /operations/research-status`;
   - export routes for research cycle, model proposal, and champion/challenger comparison JSON/XLSX.

6. Add export support:
   - write research cycle JSON/XLSX from persisted cycle/artifacts;
   - write model proposal JSON/XLSX from persisted proposal/ledger;
   - write champion-vs-challenger comparison XLSX;
   - record export metadata with source IDs, workbook sheets, and SHA-256.

7. Add tests:
   - unit tests for dry-run, stale blocking, refresh-data blocking, no silent activation, proposal approve/reject/activate guards, ledger append/query, comparison pass/block cases, operations status, and exports;
   - expand repository parity for new tables;
   - expand persisted API smoke for SQLite/Postgres research cycle/proposal surfaces;
   - add `make research-cycle-test` and `make research-status-test`.

8. Update documentation:
   - README, handoff, data model, persistence, validation, replay-aware selection, meta-scoring, model review, drift, replay-window, data-quality, API smoke, performance, and operational hardening docs;
   - add controlled-cycle, proposal-lifecycle, champion/challenger governance, decision-ledger, completion, research-cycle, and model-proposals status docs.
