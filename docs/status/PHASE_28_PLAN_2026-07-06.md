## Purpose

Phase 28 runs a pre-registered diagnostic experiment for the `trend continuation short` setup family. The objective is to test whether the weak positive Phase 22 source-replay attribution survives chronological OOS validation, portfolio replay, counterfactual replay, calibration diagnostics, and full-grid sensitivity.

This phase is not model activation. It does not approve proposals, loosen validation gates, loosen calibration gates, loosen sensitivity gates, select filters from OOS outcomes, select thresholds from OOS outcomes, use future labels, use future outcomes, use realized same-bar ambiguity as a live filter, add broker execution, add order routing, add production WebSocket ingestion, add black-box ML, or claim profitability.
