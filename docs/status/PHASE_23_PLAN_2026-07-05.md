# Phase 23 Plan - Diagnostic Candidate Filter Experiment

## Status

`PHASE_23_STATUS = IN_PROGRESS_DIAGNOSTIC_FILTER_EXPERIMENT`

Phase 23 starts from the Phase 22 accepted diagnostic rejection state and tests whether a predeclared signal-time-only filter can improve the 15min `ten_am_reversal_zone` research pocket without using future outcomes, realized same-bar ambiguity, future labels, stale bypasses, broker execution, WebSocket production ingestion, proposal approval, or model activation.

## Scope

- Target cohort: persisted 15min candidates for `SPY`, `QQQ`, `AAPL`, and `NVDA` where the persisted feature `time_bucket` is `ten_am_reversal_zone`.
- Data source: real persisted bars, features, candidates, labels, replay runs, score audits, and sensitivity rows from the evidence database only.
- Filter basis: signal-time fields available on the candidate/feature row at the signal timestamp.
- Experiment class: research diagnostic. No profitability claim, model activation, proposal approval, or production execution path is in scope.

## Required Sequence

1. Confirm evidence preflight: evidence DB role, fixture rows, active model rows, dirty windows, and Phase 22 baseline sensitivity availability.
2. Define a versioned filter spec with hash and leakage constraints.
3. Build a chronological discovery/validation split before reviewing filtered validation results.
4. Run filtered `candidate_market_replay` and `model_training_counterfactual` experiments from persisted bars/features/candidates.
5. Run full default sensitivity grid for each filtered replay.
6. Record leakage/split diagnostics, low-sample warnings, calibration/score-slice diagnostics where score audits exist, and specialist-candidate decision.
7. Export replay/filter/sensitivity/comparison artifacts with source IDs and hashes.
8. Run required tests and secret scans.

## Candidate Filters

- `P23_FILTER_A_BASE_15M_TEN_AM`: 15min `ten_am_reversal_zone` only.
- `P23_FILTER_B_AMBIGUITY_SUPPRESSED`: Filter A plus signal-time ambiguity-risk suppression using predeclared proxies only.
- `P23_FILTER_C_WEAK_FAMILY_SUPPRESSED`: Filter B plus weak setup-family suppression derived from Phase 22 diagnostics and discovery/training evidence only.
- `P23_FILTER_D_TAKE_WATCH_SLICE`: Filter A plus persisted replay-aware score action `TAKE` or `WATCH`, diagnostic only.

## Acceptance Boundary

Phase 23 can only be accepted as a diagnostic phase if real persisted data is used, filters are leakage-clean, downstream replay/sensitivity artifacts are rebuilt from those filtered candidates, exports are generated with hashes/source IDs, tests pass, and secret scans pass.

Any favorable subset remains a future specialist challenger-candidate experiment only. It is not an activated strategy.
