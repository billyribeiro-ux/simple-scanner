# Phase 21T Plan - Clean Evidence Expansion and Challenger Recovery

Status date: 2026-07-05

`PHASE_21T_PLAN_STATUS = READY`

## Scope

Phase 21T expands the clean real-data evidence base and reruns challenger recovery analysis. It is not a deployment phase and not model activation.

The source evidence database is `adaptive_market_decoder_evidence` with `AMD_DB_ROLE=evidence`. The isolated test database remains separate through `TEST_DATABASE_URL` and `AMD_DB_ROLE=test` during mutating regression checks.

## Guardrails

- Do not activate any model.
- Do not approve the rejected challenger.
- Do not loosen validation, calibration, sensitivity, review, or freshness gates.
- Do not write test fixtures to the evidence DB.
- Do not use contaminated DB rows as evidence.
- Do not add broker execution, order routing, options, gamma, Greeks, IV, market internals, Level 2, dark-pool data, order-book data, production WebSocket ingestion, black-box ML, self-learning language, or profitability claims.
- Do not expose `FMP_API_KEY` or place API keys in query strings.
