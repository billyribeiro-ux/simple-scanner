# Phase 8 Plan - Replay-Aware Baseline Model Selection

Status date: 2026-07-01

## Scope

Phase 8 adds a deterministic replay-aware baseline model-selection layer. It remains a scanner, research, replay, validation, model metadata, and export system only. It does not add broker execution, order routing, options, gamma, Greeks, IV, market internals, WebSocket, Level 2, dark-pool data, black-box ML, self-learning claims, or profitability claims.

The work starts from accepted Phase 7 replay audit infrastructure: replay provenance hashes, stale gates, explicit replay validation selection, sensitivity runs, label-vs-replay comparison, SQLite/Postgres repository parity, and reproducible exports.

## Implementation Sequence

1. Build the candidate outcome dataset module.
   - Read persisted `replay_runs`, `simulated_trades`, `candidate_signals`, feature snapshots, sensitivity runs, comparison reports, stale-window status, and validation metadata.
   - Preserve `SKIPPED` rows with skip reasons.
   - Treat overlap, portfolio limit, cooldown, duplicate, missing future bars, and stale skips as unobserved outcomes, not losses.
   - Allow invalid risk plan, insufficient context, insufficient reward/risk, regime filter, and data-quality skips to inform suppression rules without counting them as realized R outcomes.
   - Enforce training-window filters and stale replay blocking by default.

2. Build the evidence cube and hierarchy.
   - Aggregate replay outcome rows by required dimensions: symbol, side, setup type, market regime, and time bucket.
   - Add interval, day of week, relative strength/volume buckets, VWAP/opening-range/gap/trend buckets, sweep/failed-breakout flags, and same-bar ambiguity flags where feature snapshots provide them.
   - Compute sample, taken, skipped, observed outcome, expectancy, profit factor, drawdown, MFE/MAE, target/stop, ambiguity, robustness, fragility, divergence, stale-warning, lower-bound, and evidence-grade metrics.
   - Implement hierarchy/backoff from exact symbol cells through broader side-global evidence, with shrinkage so tiny samples cannot dominate.

3. Build the replay-aware meta-scorer.
   - Produce `signal_quality_score`, grade, action, expected R estimate, component scores, penalties, suppression reasons, reason codes, warning codes, evidence keys used, model version, and scoring config version.
   - Suppress critical no-evidence, insufficient-outcome, negative shrunk expectancy, weak profit factor, sensitivity fragility, stale data, same-bar ambiguity dependency, label-vs-replay divergence, critical data-quality, invalid risk, poor reward/risk, setup block, and regime block cases.
   - Keep scores explainable; do not describe them as calibrated probabilities.

4. Add persistence and migrations.
   - Add `model_evidence_cells` and `candidate_score_audits`.
   - Update SQLite initialization, SQLAlchemy schema metadata, Alembic revision `0005_phase8_replay_aware_models`, repository registry, Postgres expected schema revision, db inspection expectations, and repository parity coverage.

5. Extend training, validation, activation, and APIs.
   - Preserve existing `statistical_evidence_baseline` training.
   - Extend `/models/train` for `model_type=replay_aware_baseline`.
   - Add evidence retrieval, candidate scoring, score-audit retrieval, and `replay_aware_walk_forward` validation support.
   - Require accepted replay-aware validation before replay-aware model activation.
   - Keep `label_derived` and `candidate_market_replay` validation modes intact.

6. Integrate scanner scoring.
   - Prefer an active `replay_aware_baseline` model when present.
   - Fall back to current baseline scoring with `no_replay_aware_model_active` warning when absent.
   - Persist score audits when replay-aware scoring is used.
   - Prevent high-confidence output without replay evidence.

7. Add replay-aware exports.
   - Add replay-aware model summary XLSX, evidence cells CSV/XLSX, score audits CSV/XLSX, and replay-aware validation XLSX.
   - Persist export metadata with SHA-256 and workbook sheet names.
   - Keep exports in the ignored local exports directory and secret-free.

8. Update documentation.
   - Update README, handoff, methodology, data model, persistence, API smoke, performance, replay audit, and validation docs.
   - Add `docs/replay-aware-model-selection.md`, `docs/meta-scoring-methodology.md`, `docs/status/PHASE_8_MODEL_SELECTION_2026-07-01.md`, and `docs/status/PHASE_8_COMPLETION_2026-07-01.md`.
   - Document counterfactual replay as a next step unless it fits safely after the required persistence/model path.

9. Verify.
   - Add focused unit tests for dataset rules, evidence shrinkage/backoff, meta-scoring, training, validation, scanner fallback/active paths, exports, and repository parity.
   - Run practical gates: backend pytest, ruff, mypy, SQLite smoke, Postgres smoke when available, repository parity, replay/export tests, frontend check/build/test/lint, compileall, diff check, and secret scan.

## Guardrails

- Use only replay outcomes available inside allowed training windows.
- Do not train on validation/test replay outcomes.
- Do not weaken stale replay gates or secret handling.
- Do not write, log, export, or bundle FMP secrets.
- Keep Node target at `24.18.0` and Python target at `3.14.6` per project runtime docs.
- Do not redesign the frontend unless backend type changes force a minimal fix.
