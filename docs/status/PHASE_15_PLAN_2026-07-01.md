# Phase 15 Plan - Live FMP Entitlement and Production Data Pipeline

Date: 2026-07-01
Base commit: `fa7327db16892f2dfb156e7ab59eadf5b328f249`

## Objective

Turn FMP from a mocked provider into a safely verified REST data source for Adaptive Market Decoder V1, without exposing secrets and without adding trading execution or automatic model activation.

## Implementation Sequence

1. Confirm official FMP docs and document the stable REST/WebSocket capability matrix in `docs/research/fmp-live-entitlement-matrix.md`.
2. Add persistence for `provider_capability_checks` and `ingestion_runs` in SQLAlchemy schema, SQLite bootstrap, Alembic, repository registry, db inspection, and repository parity tests.
3. Harden `services/quant-engine/app/data/fmp.py` so all REST calls use the `apikey` header only, classify denied/rate-limited/empty/error responses, capture redacted request metadata, parse quote/quote-short/batch/EOD/intraday responses, and keep WebSocket gated as an optional probe.
4. Add a dedicated FMP pipeline service for persisted capability checks, smoke runs, quote snapshots, EOD bars, intraday bars, incremental intraday refresh, provider status, safe exports, and data coverage summaries.
5. Wire FastAPI routes for provider capability history, smoke, FMP quote/EOD/intraday ingestion, ingestion run lookup, provider status, and provider/data exports.
6. Extend data-quality reporting with source coverage, latest bar timestamps, provider request summary, ingestion run summary, and capability warnings.
7. Extend scheduler job support for bounded FMP jobs only: `fmp_capability_check`, `fmp_quote_snapshot`, `fmp_eod_refresh`, `fmp_intraday_refresh`, and `fmp_incremental_intraday_refresh`. Jobs block cleanly with `fmp_api_key_required` when the runtime key is missing.
8. Update `scripts/fmp_smoke.py` and Make targets so no-key smoke is a successful skip, mocked entitlement/ingestion tests are separate, and live smoke is explicit.
9. Add thin Svelte operator UI routes at `/operations/provider` and `/operations/data`, plus an FMP status card on `/operations`.
10. Add mocked backend tests for header-only auth, no query-string key, response parsing, capability persistence, ingestion idempotency, scheduler blocking, API routes, data quality coverage, and exports. Add Playwright smoke coverage for the new operator pages.
11. Update Phase 15 operator docs, handoff, runbooks, runtime setup, persistence docs, API smoke docs, operational hardening, performance notes, and operator UI docs.
12. Run practical backend, frontend, database, Docker, and secret-scan checks. Live FMP smoke runs only when `FMP_API_KEY` is already present in the runtime environment.

## Bounded Defaults

- Capability symbols: `SPY,QQQ,AAPL,NVDA`.
- Ingestion default symbols: `AMZN,AAPL,TSLA,SPY,QQQ,IWM,NVDA,GOOGL,BABA,SHOP`.
- REST intervals: `1min`, `5min`, `15min`.
- Max FMP scheduler symbols: 10.
- Max FMP scheduler intraday span: 5 calendar days.
- WebSocket probe: disabled unless `AMD_ENABLE_FMP_WS_PROBE=true`.

## Non-Goals

No broker execution, order routing, option analytics, market internals, Level 2, dark-pool data, order book logic, automatic model activation, automatic scanner deployment, self-learning claims, or profitability claims are in scope.
