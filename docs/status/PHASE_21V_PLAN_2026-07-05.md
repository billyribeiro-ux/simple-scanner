## Implementation Plan

1. Add bounded sensitivity metadata:
   - Record `coverage_mode`, planned/completed/remaining scenario counts, full-default-grid completion, configured-grid completion, partial disclosure, coverage warnings, and runtime.
   - Support `TIERED_ESSENTIAL` as a deterministic bounded grid.
   - Keep bounded or partial sensitivity from satisfying full-grid-required gates.

2. Improve replay runtime without changing behavior:
   - Maintain active portfolio exit timestamps directly during replay so scenario runs avoid repeated full prior-trade scans.

3. Wire sensitivity disclosure into governance:
   - Include sensitivity summaries in model review reports and exports.
   - Block required sensitivity when evidence is missing, partial, bounded instead of full-grid, incomplete, or gate-failed.
   - Keep comparison and proposal conservative through existing review readiness gates.

4. Re-run evidence operations:
   - Run evidence/test DB guardrails.
   - Confirm Phase 21U baseline health.
   - Run bounded `1min`, `5min`, and `15min` candidate-market and counterfactual sensitivity from real replay sources.
   - Re-run governance with sensitivity IDs included and `sensitivity_required=true`.
   - Run strict research-cycle dry-run with `allow_stale=false`.

5. Produce reports and exports:
   - Sensitivity profile/results.
   - Interval readiness.
   - Governance review.
   - Comparison/proposal status.
   - Completion certification with source IDs, hashes, tests, scans, fixture count, dirty windows, and active model count.
