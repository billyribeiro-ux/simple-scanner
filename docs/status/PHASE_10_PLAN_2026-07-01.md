# Phase 10 Plan - Multi-Window Replay, Calibration Drift, And Model Review

Status date: 2026-07-01

## Scope

Phase 10 adds controlled multi-window replay orchestration, calibration drift reporting, operational model-review reports, data-quality reporting, export reproducibility checks, and safe Postgres query diagnostics. It does not add broker execution, order routing, options/Greeks/IV, WebSocket scope, market internals, black-box ML, self-learning behavior, frontend redesign, or profitability claims.

## Implementation Sequence

1. Add persistent tables and repositories for `replay_window_sets`, `replay_window_results`, `model_calibration_drift_reports`, `model_calibration_drift_windows`, and `model_review_reports` through SQLite bootstrap, SQLAlchemy schema, Alembic revision `0007_phase10_model_review`, db-inspect expectations, and repository parity.
2. Create a window-generation module that supports `daily`, `rolling`, `anchored`, and `custom` modes with explicit train/validation/test/replay boundaries, embargo minutes, max-window guardrails, and insufficient-data/stale-window warnings.
3. Add a bounded orchestration service that creates, lists, gets, runs, and exports replay window sets synchronously for V1. It will reuse persisted bars/features/candidates/replay services and persist each window result with replay IDs, calibration IDs, comparison IDs, metrics, warnings, and status.
4. Add `app/models/calibration_drift.py` to aggregate persisted calibration audits and window results into rank-correlation, monotonicity, TAKE-vs-WATCH, high-grade expectancy, sensitivity, drawdown, stability, drift flags, and severity. Persist report summaries plus optional per-window rows.
5. Add model-review report service and persistence that combines model provenance, validation, calibration, drift, sensitivity, comparisons, stale-window/data-quality status, export status, and unresolved warnings into diagnostic readiness statuses `PASS`, `WATCH`, `REVIEW`, or `BLOCK`. The report never activates or deactivates a model.
6. Add `GET /data/quality-report` to summarize persisted bars, missing/duplicate/invalid bars, stale feature/candidate/label/replay windows, provider request metadata, and recommended rebuild steps without exposing secrets.
7. Add API routes for replay window sets, calibration drift, model review reports, exports, and data quality.
8. Extend exports for replay window-set XLSX, calibration drift XLSX/JSON/window rows, and model review XLSX/JSON. Each export records file SHA-256, row count, workbook sheets, source IDs, and non-secret config/provenance.
9. Add `scripts/db_query_diagnostics.py` hardening if needed and `make db-query-diagnostics`. Remove password-shaped local database URLs from diagnostics by assembling dev URLs from non-secret local components.
10. Update API smoke for a small controlled window-set orchestration, drift report, model review report, data-quality report, and new exports on SQLite and Postgres.
11. Add focused tests for window generation/orchestration, calibration drift flags/severity, model review readiness, data-quality reporting, export reproducibility, and repository parity.
12. Update README, HANDOFF, data model, persistence, API smoke, performance, replay/calibration docs, operational hardening, and new Phase 10 methodology/status docs.
