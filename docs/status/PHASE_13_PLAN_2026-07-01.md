# Phase 13 Plan

Status date: 2026-07-01

Phase 13 adds local operator reliability around Docker/Postgres recovery, runbooks, and a bounded non-autonomous scheduler for research-cycle preparation only. It must not add broker execution, order routing, automatic proposal approval, automatic scanner model activation, live FMP production ingestion, options data, WebSocket data, self-learning claims, or profitability claims.

## Implementation Sequence

1. Document recovery and operator procedure:
   - create `docs/local-operator-runbook.md`;
   - create `docs/docker-postgres-troubleshooting.md`;
   - create `docs/operator-daily-procedure.md`;
   - create `docs/non-autonomous-scheduler.md`;
   - update runtime, handoff, operations, API smoke, data quality, performance, governance, and UI docs.

2. Add scheduler persistence:
   - add Alembic revision `0009_phase13_scheduler`;
   - add `scheduler_jobs` and `scheduler_job_events` to SQLAlchemy schema;
   - add SQLite bootstrap tables and indexes;
   - update expected table/revision/index/JSON checks in `scripts/inspect_db_schema.py`;
   - add repository methods for create, get, list/filter, status update, cancel, append/list events, and status summary.

3. Add scheduler service:
   - implement `SchedulerService` for bounded synchronous execution;
   - support `research_cycle_dry_run`, `research_cycle_run`, `data_quality_report`, `export_research_cycle`, and `export_operations_status`;
   - default `max_jobs` conservatively;
   - block `refresh_data=true` when `FMP_API_KEY` is missing without attempting provider access;
   - never call proposal approve/reject/activate, model activation, scanner activation, broker, or order-routing code;
   - persist every status transition and event without secrets.

4. Add API routes:
   - `POST /scheduler/jobs`;
   - `GET /scheduler/jobs`;
   - `GET /scheduler/jobs/{job_id}`;
   - `POST /scheduler/jobs/{job_id}/run`;
   - `POST /scheduler/jobs/run-pending`;
   - `POST /scheduler/jobs/{job_id}/cancel`;
   - `GET /scheduler/jobs/{job_id}/events`;
   - `GET /operations/scheduler-status`.

5. Add operator UI:
   - add typed shared schemas and frontend API helpers;
   - add `/operations/scheduler` list/status page;
   - add `/operations/scheduler/[job_id]` detail page;
   - add a small scheduler status section to `/operations`;
   - keep controls limited to create safe job, run pending, run one queued job, and cancel queued job;
   - do not add auto-run toggles, activation controls, or trading controls.

6. Add tests:
   - scheduler repository persistence and reinitialization tests;
   - scheduler service transition, FMP gating, run-pending bounds, cancel, and no-activation tests;
   - scheduler API route tests;
   - docs/runbook regression tests;
   - frontend e2e coverage for scheduler list/detail/create/run-pending and no activation/trading controls;
   - update persisted API smoke and repository parity where practical.

7. Add Make/CLI helpers:
   - `make scheduler-test`;
   - `make scheduler-status`;
   - optional `scripts/scheduler_status.py`;
   - keep output non-secret.
